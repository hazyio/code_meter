# Security Policy

## Reporting a Vulnerability

If you discover a security issue in Commit — particularly anything involving how the WakaTime API key is stored, transmitted, or exposed — please **do not open a public issue**.

Instead, report it privately via [GitHub's private vulnerability reporting](https://github.com/hazyio/commit/security/advisories/new) on this repository, or contact the maintainer directly through the contact info on their GitHub profile.

Please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, if possible
- Any suggested fix, if you have one

We'll acknowledge reports as quickly as possible and keep you updated as the issue is investigated and resolved. Once a fix is released, we're happy to credit reporters in the release notes, if you'd like.

## Supported Versions

As a young, actively developed project, only the latest released version is supported with security fixes.
